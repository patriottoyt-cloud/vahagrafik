# Ваха Лавка · График смен

Веб-приложение для сбора пожеланий официантов и автосборки графика.
Это самостоятельный сайт (не привязан к Claude) — данные хранятся в Supabase,
поэтому график виден и сохраняется у всех, кто открывает ссылку, с любого телефона.

## Шаг 1. Supabase (хранилище данных) — 5 минут

1. Зайти на [supabase.com](https://supabase.com), создать бесплатный аккаунт и новый проект (New project).
2. Дождаться, пока проект поднимется (около минуты).
3. Слева — **SQL Editor** → **New query**. Вставить содержимое файла `supabase_setup.sql` из этой папки и нажать **Run**. Это создаст одну таблицу `app_kv`, где будет храниться всё: список официантов, период, пожелания, готовый график.
4. Слева — **Settings** → **API**. Скопировать оттуда:
   - **Project URL** (похоже на `https://xxxxxxxx.supabase.co`)
   - **anon public** ключ (длинная строка)

## Шаг 2. Проверить локально (необязательно, но полезно)

```bash
npm install
cp .env.example .env
```

Открыть `.env` и вставить туда свои значения из Supabase:

```
VITE_SUPABASE_URL=https://xxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=вставленный_ключ
```

```bash
npm run dev
```

Откроется адрес вроде `http://localhost:5173` — можно потыкать на компьютере перед публикацией.

## Шаг 3. Netlify (публикация сайта)

### Вариант А — просто перетащить файлы (без GitHub, самый быстрый)

1. Убедиться, что в `.env` вписаны реальные значения Supabase (шаг 2).
2. Собрать сайт:
   ```bash
   npm run build
   ```
   Появится папка `dist` — это и есть готовый сайт.
3. Зайти на [app.netlify.com/drop](https://app.netlify.com/drop) и перетащить папку `dist` в браузер.
4. Netlify выдаст ссылку вроде `https://random-name-123.netlify.app` — это и есть адрес приложения. Можно поменять на более понятный в **Site settings → Change site name**.

Минус этого способа: если что-то поменяется в коде, придётся собирать и перетаскивать заново.

### Вариант Б — через GitHub (удобно, если планируешь дорабатывать)

1. Залить эту папку в новый репозиторий на GitHub.
2. На [app.netlify.com](https://app.netlify.com) → **Add new site → Import an existing project** → выбрать репозиторий.
3. Настройки сборки Netlify подхватит сам из `netlify.toml`:
   - Build command: `npm run build`
   - Publish directory: `dist`
4. До первого деплоя — **Site settings → Environment variables** → добавить:
   - `VITE_SUPABASE_URL` = Project URL из Supabase
   - `VITE_SUPABASE_ANON_KEY` = anon public ключ из Supabase
5. **Deploy site**. Дальше при каждом пуше в GitHub сайт будет обновляться сам.

## Важно про доступ

Ссылку на сайт (и на админку внутри него — вкладка «Я администратор») видит любой,
у кого есть эта ссылка: отдельного входа/пароля нет. Для внутреннего инструмента
с закрытой ссылкой этого обычно достаточно, но саму ссылку официантам стоит давать
только на официантскую часть, а администраторскую — держать при себе.

## Структура проекта

```
src/
  App.jsx            — всё приложение (роли официант/администратор, алгоритм графика)
  storage.js          — работа с Supabase (замена служебного хранилища Claude)
  supabaseClient.js    — подключение к Supabase по ключам из .env
  main.jsx             — точка входа React
supabase_setup.sql      — SQL для разового запуска в Supabase
netlify.toml             — настройки сборки для Netlify
```
